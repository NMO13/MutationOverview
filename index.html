Dataset

The dataset consists of cancer data which is part of the Glioblastoma multiforme project (GBM-US) from the mutations API endpoint of the ICGC Data Coordination Center. The data are accessible in JSON format and can be accessed by querying the API endpoint. And example query to receive the first 100 mutations would be: https://dcc.icgc.org/api/v1/projects/GBM-US/mutations?field=id,mutation,type,chromosome,start,end&size=100&order=desc

User Goals

The goals is to provide a browser-based JavaScript tool for interactive visualization of mutations in cancer. The user should be able to see and interact with two different kinds of overviews:
  1. Type Overview: A visualization that shows the number of mutations for each mutation type.
  2. Chromosome Overview: A visualization that shows the distribution of mutations across the 22 human autosome pairs plus the two sex chromosomes.
Further, the user should be able to filter the data. Selecting a type or chromosome should filter the overall list of mutations to the subset that pass the filter and update the other view (e.g., if chromosome 17 is selected, the Type Overview should only show the data for mutations on chromosome 17). Selecting an entry in both the Chromosome Overview and the Type Overview should be allowed and be treated like a logical AND. It should be possible to clear the filters.

Solution

We use barcharts for the type overview and the mutation overview. This allows the user to get a quick visual feedback of the number and distributions of mutations. By clicking on a bar of the overview barchart, the mutation overview adapts its visualization and vice versa.

Implementation

We implemented our visualization using TypeScript an D3. We created a dataservice which retrieves the mutation data from the mutation API. The dataservice makes an AJAX call to the datapoint and supplies the data in chunks of 100 mutations or all data at once. This depends on the used selection method. The barchart and stacked barchart are implemented in separate classes. For both charts, we implemente a render method which will be called for the initial setup of the charts and each time the visualization state changes. Further, a reset button was implemented to reset the filters.

Improvements

  • It should be possible to click on the legend fields after a filter has been applied.
  • Only the first 10 pages of the dataset are visualized. It shoud be possible to click on the "next arrow" to show more pages
  • The dataset contains much more information which would be interesting to visualize. Currently, only the mutations are visualized.
